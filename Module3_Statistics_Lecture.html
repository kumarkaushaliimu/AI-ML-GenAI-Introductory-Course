📊 Sampling Strategies for Real World

🎲 Simple Random

Method: Random.choice() from population

Pros: Unbiased, statistically sound

Cons: Need complete population list

✓ Best for most ML applications

🏗️ Stratified

Method: Sample from each subgroup proportionally

Pros: Ensures representation

Cons: Need strata information

✓ Critical for imbalanced datasets

🔄 Cluster

Method: Randomly select entire clusters

Pros: Cost-effective

Cons: Less precise than random

✓ Use when population is grouped

import pandas as pd import numpy as np # Simple Random Sampling sample = df.sample(n=1000, random_state=42) # Stratified Sampling (for imbalanced classes) sample = df.groupby('target', group_keys=False).apply( lambda x: x.sample(frac=0.2) )

📏 Measures of Central Tendency

📊 Mean (μ)

Average value

Formula: μ = ΣX / N

✓ Uses all data

✗ Sensitive to outliers

Example: [10, 20, 30, 40, 50]
Mean = 30

📍 Median

Middle value

Formula: Sorted[N/2]

✓ Robust to outliers

✗ Ignores some data

Example: [10, 20, 30, 40, 50]
Median = 30

🎯 Mode

Most frequent value

Formula: argmax(count)

✓ Works with categories

✗ May not be unique

Example: [10, 20, 20, 30, 30, 30]
Mode = 30

📊 Spread & Shape: Variance, Skewness, Kurtosis

📏 Measures of Dispersion

  • Range: Max - Min (simple but crude)
  • Variance (σ²): Average squared deviation from mean
  • Std Dev (σ): √Variance (same units as data)
  • Coefficient of Variation (CV): σ/μ (compare across different scales)
  • Interquartile Range (IQR): Q3 - Q1 (robust measure)
σ² = Σ(X - μ)² / N

🔄 Distribution Shape

  • Skewness (γ₁): Measures asymmetry
    • = 0: Symmetric
    • > 0: Right-skewed (long right tail)
    • < 0: Left-skewed (long left tail)
  • Kurtosis (γ₂): Measures tailedness
    • = 3: Normal distribution
    • > 3: Heavy tails (leptokurtic)
    • < 3: Light tails (platykurtic)

🔗 Understanding Variable Relationships

📊 Covariance

Definition: How two variables vary together

Cov(X,Y) = Σ(X - μₓ)(Y - μᵧ) / N
  • > 0: Variables move together
  • < 0: Variables move opposite
  • ≈ 0: No linear relationship

📈 Correlation (r)

Definition: Standardized covariance [-1, +1]

r = Cov(X,Y) / (σₓ × σᵧ)
  • +1: Perfect positive correlation
  • 0: No correlation
  • -1: Perfect negative correlation
⚠️ Important: Correlation ≠ Causation! Two variables can be correlated without one causing the other.

🎲 Probability Foundations

📐 Probability Rules

  • 0 ≤ P(A) ≤ 1 (Always between 0 and 1)
  • P(not A) = 1 - P(A) (Complement rule)
  • P(A or B) = P(A) + P(B) - P(A and B)
  • P(A and B) = P(A) × P(B|A) (Conditional)
Key Insight: Probability is the foundation of all statistical inference. All hypothesis tests produce p-values, which are probabilities.

🎯 Common Probability Misconceptions

  • ❌ Gambler's Fallacy: Past doesn't guarantee future outcomes
  • ❌ Law of Averages: Can't "correct" random variation
  • ❌ Base Rate Fallacy: Must consider prior probability
  • ❌ Regression to Mean: Extreme values tend to become average

🔔 Probability Distributions Landscape

🎲 Discrete Distributions

  • Bernoulli: Single trial (Success/Failure)
  • Binomial: n independent trials
  • Poisson: Events in fixed interval

🔔 Continuous Distributions

  • Normal: Bell curve (most common)
  • Exponential: Waiting times
  • Uniform: Equal probability

🔔 Normal Distribution: Most Important!

📊 Key Properties

  • Bell-shaped and symmetric
  • Mean = Median = Mode
  • Defined by μ (mean) and σ (std dev)
  • 68-95-99.7 Rule:
    • 68% data within ±1σ
    • 95% data within ±2σ
    • 99.7% data within ±3σ

🎯 Why It Matters for ML

  • Central Limit Theorem: Sample means are normally distributed!
  • Foundation for confidence intervals
  • Assumption in many ML algorithms
  • Used in hypothesis testing
  • Enables prediction intervals
Many ML algorithms (linear regression, logistic regression, neural networks) perform better when features are normally distributed.

🎯 Central Limit Theorem: The Magic Theorem

For ANY population distribution with finite mean & variance:

The distribution of sample means ≈ Normal(μ, σ/√n)

This holds true regardless of the original population's shape!

✨ Why This Changes Everything

  • Justifies normal distribution assumption
  • Enables hypothesis testing
  • Allows confidence interval construction
  • Sample size ≥ 30 usually sufficient
  • Works for ANY population!

📈 Practical Implications

  • Larger samples → better approximation
  • Sample mean ≈ Population mean
  • Standard error = σ/√n
  • Precision improves with √n
  • Foundation of ALL inference!

📏 Confidence Intervals: Range Estimation

What is a Confidence Interval?

A range of plausible values for an unknown population parameter, calculated from sample data.

CI = x̄ ± zα/2 × (σ/√n)

For 95% confidence: z = 1.96 | For 99% confidence: z = 2.576

🔢 Example Calculation

  • Sample mean: 102.5
  • Std dev: 15
  • Sample size: 100
  • Standard error: 15/√100 = 1.5
  • Margin of error: 1.96 × 1.5 = 2.94
  • 95% CI: [99.56, 105.44]

✅ Correct Interpretation

  • We're 95% confident the true mean falls in this range
  • If we repeated 100 times, ~95 intervals contain μ
  • NOT "95% chance the parameter is in this range"
  • Wider intervals = more uncertainty
  • Narrower intervals = more precision

❌ Common Mistakes

  • "95% of data falls here" (WRONG)
  • "We're 95% sure the sample mean is in the interval" (WRONG)
  • Treating CI as a prediction interval
  • Assuming parameter is random
  • Using without stating confidence level

📌 Key Concepts

  • H₀ (Null): Default assumption (no effect)
  • H₁ (Alternative): What we try to prove
  • α (Alpha): Significance level (typically 0.05)
  • p-value: Probability of data given H₀ is true
  • Decision: Reject H₀ if p-value < α

⚠️ Type I & II Errors

Reject H₀ Fail to Reject
H₀ True Type I (α) Correct ✓
H₁ True Correct ✓ Type II (β)

⚖️ Choosing the Right Statistical Test

Decision Tree for Test Selection

📊 Comparing Means

  • 1 Sample vs Population: Z-test (n>30) or t-test (n<30)
  • 2 Independent Samples: Independent t-test
  • 2 Paired Samples: Paired t-test
  • 3+ Groups: ANOVA (Analysis of Variance)

🏷️ Comparing Categories

  • 2 Categorical Variables: Chi-square test
  • Goodness of Fit: Chi-square test
  • Independence Test: Chi-square test
  • Multiple Categories: Fisher's exact test

⚖️ Z-Test vs T-Test Comparison

Aspect Z-Test T-Test
Population σ Known Unknown (use sample s)
Sample Size n ≥ 30 n < 30
Distribution Standard normal Student's t (fatter tails)
Critical Value ±1.96 (95% CI) ±2.045-∞ (depends on df)
Formula z = (x̄ - μ₀) / (σ/√n) t = (x̄ - μ₀) / (s/√n)
Use When Large samples, σ known Small samples, typical ML
from scipy import stats # Z-Test z_stat = (sample_mean - pop_mean) / (pop_std / np.sqrt(n)) p_value = 2 * (1 - stats.norm.cdf(abs(z_stat))) # T-Test (more common in practice) t_stat, p_value = stats.ttest_1samp(sample, popmean=pop_mean)

💼 Real-World: A/B Testing for Conversions

📊 E-commerce Checkout Button Color Test

🔴 Control (Red Button)

Conversions: 42 / 1000

CR: 4.2%

🟢 Treatment (Green Button)

Conversions: 51 / 1000

CR: 5.1%

📈 Results

Uplift: +21.4%

Abs Diff: +0.9 pp

🔬 Statistical Test (Chi-Square)

  • H₀: Conversion rates are equal
  • H₁: Conversion rates differ
  • χ² statistic: 2.834
  • p-value: 0.092
  • Result: NOT significant at α=0.05
⚠️ The 0.9pp uplift, while practical, isn't statistically significant. Need larger sample!

💡 Key Lessons

  • Practical significance ≠ Statistical significance
  • Report effect size, not just p-value
  • Sample size calculation before experiment
  • Consider business impact & cost
  • Document assumptions & limitations

⚠️ Common Statistical Traps to Avoid

🚨 P-Hacking

What: Testing multiple hypotheses until finding significance

Problem: α = 1 - (1-0.05)^k with k tests

Solution: Pre-register hypotheses before analysis

⛓️ Correlation ≠ Causation

What: Assuming X→Y because r = 0.9

Problem: Could have confounding variables

Solution: Design controlled experiments

📊 Multiple Comparisons

What: Running many tests without adjustment

Problem: Family-wise error rate inflates

Solution: Bonferroni correction, FDR

📈 Ignoring Effect Size

What: Reporting p < 0.05 as "success"

Problem: Statistically significant ≠ Practically important

Solution: Always report Cohen's d, r-squared

🎲 Low Statistical Power

What: Using β = 0.20 (80% power)

Problem: High chance of Type II error

Solution: Power analysis before experiment

📉 Data Dredging

What: Exploring data after seeing patterns

Problem: Leads to false discoveries

Solution: Separate exploratory & confirmatory
