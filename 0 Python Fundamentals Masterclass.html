📊 Collections Comparison

Lists, Tuples, Sets, and Dictionaries

Visual comparison of Python data structures
List: Ordered, mutable, duplicates allowed. Use for data sequences.
Tuple: Ordered, immutable, faster. Use for constants & dictionary keys.
Set: Unordered, unique elements. Use for membership testing.
Dict: Key-value pairs, mutable. Use for fast lookups.

🔒 Tuples: Immutable Sequences

Ordered collections that cannot be modified

# Creating tuples empty = () single = (1,) # Note comma! coordinates = (10, 20) mixed = (1, "hello", 3.14) # Accessing (like lists) coordinates[0] # 10 coordinates[-1] # 20 # Tuple unpacking x, y = coordinates # x=10, y=20 # Fixed size - cannot modify! # coordinates[0] = 5 # ERROR!

When to Use Tuples

  • Dictionary keys: Lists can't be keys
  • Return multiple: Functions return multiple values
  • Constants: Immutable data
  • Performance: Faster than lists

🎯 Sets: Unordered, Unique Collections

Fast membership testing and duplicate removal

# Creating sets numbers = {1, 2, 3, 4, 5} duplicates = {1, 1, 2, 2, 3} # {1, 2, 3} - duplicates removed! # Set operations A = {1, 2, 3, 4} B = {3, 4, 5, 6} print(A | B) # {1, 2, 3, 4, 5, 6} - Union print(A & B) # {3, 4} - Intersection print(A - B) # {1, 2} - Difference

Set Methods

  • .add(item)
  • .remove(item)
  • .union(other)
  • .intersection(other)
  • .difference(other)

Best Uses

  • Remove duplicates from list
  • Fast membership testing
  • Mathematical operations (union, intersection)
  • Finding unique elements

✨ Comprehensions: Pythonic Way

Write concise, readable code with comprehensions

List Comprehensions

# Basic squares = [x**2 for x in range(5)] # [0, 1, 4, 9, 16] # With condition evens = [x for x in range(10) if x % 2 == 0] # [0, 2, 4, 6, 8] # Nested matrix = [[i+j for j in range(3)] for i in range(3)] # [[0,1,2],[1,2,3],[2,3,4]]

Dictionary Comprehensions

# Basic squares = {x: x**2 for x in range(5)} # {0:0, 1:1, 2:4, 3:9, 4:16} # With condition even_squares = {x: x**2 for x in range(10) if x % 2 == 0} # {0:0, 2:4, 4:16, 6:36, 8:64} # From lists keys = ['a', 'b', 'c'] values = [1, 2, 3] d = {k:v for k,v in zip(keys, values)}

⚠️ Common Python Mistakes

Avoid these errors in your code

❌ Mistakes to Avoid

  • Indentation: Python is whitespace-sensitive
  • Type mismatch: "5" + 5 causes error
  • Missing colon: if, for, while need :
  • Out of range: list[10] if only 5 items
  • Mutable defaults: def func(list=[])
  • String immutability: Cannot modify strings
# WRONG: Indentation error if x > 5: print("too big") # Not indented! # RIGHT: Proper indentation if x > 5: print("too big") # WRONG: Type error result = "5" + 5 # RIGHT: Convert type result = int("5") + 5 # WRONG: Missing colon if x > 5 print(x) # RIGHT: Colon required if x > 5: print(x)

✅ Python Best Practices

Write clean, professional code

✓ Do This

  • Use meaningful variable names
  • Add comments explaining logic
  • Use list comprehensions
  • Follow PEP 8 style guide
  • Keep functions small & focused
  • Use f-strings for formatting

✗ Don't Do This
